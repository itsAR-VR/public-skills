# Hormozi Skill: Closing

Use when: sales calls stall, objections dominate, or close rates are inconsistent.

## Core principles
- Closing is a process, not improvisation.
- Most “infinite” objections reduce to a small set of buying objections.
- Objection handling must loop back to value/offer fit.
- Follow-up is part of closing, not separate admin work.
- Volume and consistency matter (“volume negates luck” operating mindset).

## Key frameworks
### 1) The 5 Buying Objections
1. **Time** (not enough time / wrong priority)
2. **Money** (price/cost concerns)
3. **Decision-Maker** (needs partner/boss approval)
4. **Preference** (wanted different approach)
5. **Stall** (“need to think about it” / delay)

### 2) Objection Looping Framework
From handbook language: acknowledge objection, isolate true objection (triage), reconfirm value, loop to decision.

Practical sequence:
1. Acknowledge and label concern.
2. Ask triage question to isolate root issue.
3. Reconnect desired outcome/problem cost.
4. Reframe with offer fit.
5. Ask for next commitment (buy or concrete next step).

### 3) BAMFAM
**Book A Meeting From A Meeting** when deal can’t close now. Never leave with vague “we’ll follow up.”

### 4) No-show / follow-up discipline
Use predefined scripts and timed follow-ups; keep one objective per message: rebook, close, or set a clear timeline.

## Practical templates
### Template A: Objection isolate question
`Totally fair. Just so I can help correctly—if we solved [stated objection], would you move forward today?`

### Template B: Money loop
`Compared to [cost of inaction], this investment is [context]. If we can structure terms as [option], are you ready to start?`

### Template C: BAMFAM line
`Makes sense. Let’s lock the exact next time now so this doesn’t slip—does [option A] or [option B] work better?`

## Examples
1. Time objection:
- Reframe as priority; remove low-value actions; commit to first implementation step now.

2. Decision-maker objection:
- Book next call with both parties present; send recap + outcome model before call.

3. Stall objection:
- Walk decision framework; quantify cost of waiting; set decision deadline.
