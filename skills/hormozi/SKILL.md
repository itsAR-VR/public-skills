# Hormozi Skill Pack

Action-oriented operating references distilled from Hormozi summaries/text extracts.

## When to use which file

- **offer-construction.md**
  - Use when creating/reworking offers, increasing conversion without price cuts.
  - Covers: Value Equation, Grand Slam Offer build flow, Trim & Stack, scarcity/urgency/guarantees/naming.

- **lead-generation.md**
  - Use when pipeline is inconsistent or lead volume/quality is weak.
  - Covers: Core Four methods, lead magnet design, Hook-Retain-Reward content, referrals/affiliates.

- **advertising.md**
  - Use when ads get poor response or creative production is chaotic.
  - Covers: hook mechanics, 70-20-10 hook allocation, Hook-Meat-CTA assembly line, awareness-based hooks, ad framework bank.

- **closing.md**
  - Use when sales calls stall or objection handling is inconsistent.
  - Covers: 5 buying objections, objection looping, BAMFAM follow-up, no-show/follow-up discipline.

- **retention.md**
  - Use when churn rises or LTV plateaus.
  - Covers: 5 Horsemen of retention, churn checklist workflow, engagement triggers, exit interview saves.

- **pricing.md**
  - Use when margin/profit is weak, or you need structured price changes.
  - Covers: 3 pricing models, total-return pricing metrics, anchoring, annual/prepay logic, RAISE price-raise communication, fast-cash cadence.

- **proof.md**
  - Use when trust is the conversion bottleneck.
  - Covers: 13-point proof checklist hierarchy, testimonial structure, proof capture and deployment ops.

- **hormozi-humanizer**
  - Use when a memo, post, client update, or founder note sounds like AI or
    corporate filler.
  - Covers: plain-English point recovery, anti-slop phrase cleanup, and
    preserving the user's actual bullets instead of replacing their thinking.

## Notes
- Content is derived from extracted Hormozi source text in the pipeline directory.
- Some markdown summary files were thin/scaffold-only; extraction text files were used for framework detail where needed.
