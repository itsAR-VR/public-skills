# Website Discovery And SEO Contract

Research date: 2026-05-18

Purpose: make websites and public pages findable, understandable, shareable,
and previewable. A page that looks premium but cannot be found, scanned, linked,
or summarized is not finished.

Use this for landing pages, public product pages, docs pages, content pages,
marketing pages, blogs, help centers, directories, and any website route that
could appear in search or social previews.

## Design Rules

- The first viewport must communicate the page's real offer, product, place,
  person, or object.
- Headings should match how a real user scans the page, not how the code is
  organized.
- Search engines are not the only audience. A person pasting the link in Slack,
  email, or social media should get a clear preview.
- Do not bury essential content inside images, animations, carousels, or
  canvas-only visuals.
- For docs/help pages, make search, section labels, and next steps obvious.
